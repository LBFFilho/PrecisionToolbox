Devloper’s Guide

For Developers

For Developers

In order to better understand the internals of how ADI's transceiver and High-Speed Converter Toolbox system objects are implemented, it is also useful to understand the overall structure of MATLAB's libiio package and how adi.common serves as the interface layer on which both of these toolboxes rely.

matlabshared.libiio

First, let us take a look at the list of superclasses that the base class in MATLAB's libiio package inherits from.
superclasses('matlabshared.libiio.base')
Superclasses for class matlabshared.libiio.base: matlab.System handle matlab.mixin.CustomDisplay matlab.system.mixin.Propagates matlabshared.libiio.context matlabshared.libiio.channel matlabshared.libiio.device matlabshared.libiio.buffer matlabshared.libiio.top
As shown by the output displayed, the matlabshared.libiio.base is a child class of the remaining classes that get installed when MATLAB's Communications Toolbox Support Package for Xilinx Zynq Based Radio. Additionally, since this class is the parent for the classes in the adi.common interface layer, we can also look at the abstract properties and methods that the child classes of matlabshared.libiio.base are expected to implement.
meta.abstractDetails('matlabshared.libiio.base')
Abstract methods for class matlabshared.libiio.base: releaseChanBuffers % defined in matlabshared.libiio.base configureChanBuffers % defined in matlabshared.libiio.base Abstract properties for class matlabshared.libiio.base: SamplesPerFrame % defined in matlabshared.libiio.base devName % defined in matlabshared.libiio.base kernelBuffersCount % defined in matlabshared.libiio.base dataTypeStr % defined in matlabshared.libiio.base channelCount % defined in matlabshared.libiio.base DataTimeout % defined in matlabshared.libiio.base isOutput % defined in matlabshared.libiio.base
When we look at the properties list for adi.common.RxTx class,
properties('adi.common.RxTx')
Properties for class adi.common.RxTx: EnabledChannels uri enIO SamplesPerFrame devName kernelBuffersCount dataTypeStr
we can notice that all the properties have been inherited by this class. Note however that this class is still an abstract class which defines other abstract properties and methods which its child classes must implement. So, it cannot be instantiated as shown below.
% c = adi.common.RxTx % produces an error
These abstract properties and methods can be listed as follows:
meta.abstractDetails('adi.common.RxTx')
Abstract methods for class adi.common.RxTx: setupInit % defined in adi.common.RxTx Abstract properties for class adi.common.RxTx: Type % defined in adi.common.RxTx ComplexData % defined in adi.common.RxTx SamplesPerFrame % defined in matlabshared.libiio.base devName % defined in matlabshared.libiio.base kernelBuffersCount % defined in matlabshared.libiio.base dataTypeStr % defined in matlabshared.libiio.base isOutput % defined in matlabshared.libiio.base
Before, we look at how these abstract properties and methods are implemented by the Transceiver Toolbox and High-Speed Converter Toolbox system objects, we look at the adi.common interface layer in some more detail. As an exercise, running the superclasses() function with each of the classes available in adi.common package as the argument will reveal the superclasses of that class.
